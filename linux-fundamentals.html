Connecting via SSH
01 / 10

SSH (Secure Shell) lets you remotely access target machines securely over a network. This is the standard way to connect to HTB lab machines.

TERMINAL
$ ssh htb-student@<TARGET_IP>
htb-student@<TARGET_IP>'s password: ••••••••
Welcome to Ubuntu 20.04 LTS...
htb-student@JohnDoe:~$
💡
TipReplace <TARGET_IP> with the actual IP shown in your HTB lab. The password is provided in the module instructions.

ip — Network Interfaces & MTU Quiz Gap
05 / 10

The ip command is used to show or manipulate routing, network devices, interfaces, and tunnels. The key subcommands for enumeration are ip a (addresses) and ip link (link-layer info).

The MTU (Maximum Transmission Unit) is a property of each network interface — it defines the largest packet size that can be transmitted. The default for Ethernet is 1500 bytes. Finding which interface has MTU 1500 is a common recon/quiz task.

TERMINAL — ip a (show all interfaces)
JohnDoe@htb[~]$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN
link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
inet 127.0.0.1/8 scope host lo
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast
link/ether 00:0c:29:xx:xx:xx brd ff:ff:ff:ff:ff:ff
inet 10.129.x.x/16 scope global eth0
TERMINAL — filter MTU lines only (faster)
JohnDoe@htb[~]$ ip a | grep mtu
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 ...
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ...

# Or use ip link for link-layer only:
JohnDoe@htb[~]$ ip link
★
How to answer the MTU quiz question The question asks for the name of the interface with MTU 1500. Look at the line containing mtu 1500 — the interface name is the word before the colon on that line.

2: eth0: ... mtu 1500 → answer is eth0
3: ens33: ... mtu 1500 → answer is ens33
CommandWhat it shows
ip aAll interfaces with IP addresses, MTU, and state
ip addrSame as ip a (full form)
ip linkLink-layer info (MAC, MTU, flags) — no IP addresses
ip a | grep mtuShow only MTU lines for all interfaces
ip routeRouting table — useful for finding gateways

ls -i — Inode (Index) Numbers Quiz Gap
06 / 10

Every file in Linux has an inode (index node) — a unique number that the filesystem uses to identify it, regardless of filename. This wasn't covered in the main notes but appeared as a quiz question.

The -i flag on ls adds the inode number as a column to the left of each file name. ls -la alone does not show inodes — you need -i specifically.

TERMINAL — ls -i (all files in current dir)
JohnDoe@htb[~]$ ls -i
262401 sudoers 131074 passwd 655362 shadow

# Target a specific file:
JohnDoe@htb[~]$ ls -i sudoers
262401 sudoers
TERMINAL — combine with -la for full detail
JohnDoe@htb[~]$ ls -ila
262401 -r--r----- 1 root root 669 Jan 19 2021 sudoers
131074 -rw-r--r-- 1 root root 1822 Jan 19 2021 passwd
★
How to answer the inode quiz question Navigate to the directory containing the file, then run ls -i <filename>. The number on the left is the inode (index number).

262401 sudoers → answer is 262401
CommandWhat it shows
ls -iInode number for all files in current directory
ls -i <file>Inode number for a specific file
ls -ilaInode + long format + hidden files (maximum detail)
stat <file>Full inode info including block count, timestamps, and links
⚠️
Why ls -la doesn't show itls -la shows permissions, owner, size, and timestamps — but not the inode. You need the -i flag specifically for that.

uname — Kernel & System Info
07 / 10

uname prints OS information. The most useful flags are -a (all info) and -r (kernel release — for finding kernel exploits).

TERMINAL
JohnDoe@htb[~]$ uname -a
Linux box 4.15.0-99-generic #100-Ubuntu SMP Wed Apr 22 20:32:56 UTC 2020 x86_64 GNU/Linux

JohnDoe@htb[~]$ uname -r
4.15.0-99-generic
FlagOutput
uname -aAll info (kernel, hostname, release, version, arch, OS)
uname -rKernel release — use to search for kernel exploits
uname -sKernel name (e.g. Linux)
uname -nNetwork node hostname
uname -mMachine hardware (e.g. x86_64)
uname -oOS (e.g. GNU/Linux)
💡
Kernel exploit tipRun uname -r, then search "4.15.0-99-generic exploit" to quickly find known kernel vulnerabilities for privilege escalation.

Navigation — pwd, ls, cd
08 / 10

Navigation in Linux is done entirely through the terminal. Confirm where you are with pwd, list contents with ls, and move with cd.

pwd — Print Working Directory
TERMINAL
JohnDoe@htb[~]$ pwd
/home/JohnDoe
ls — List Directory Contents

Desktop Documents Downloads Music Pictures Public Templates Videos

JohnDoe@htb[~]$ ls -la

cd — Change Directory
TERMINAL — cd variants
# Navigate to absolute path
JohnDoe@htb[~]$ cd /dev/shm
JohnDoe@htb[/dev/shm]$

# Jump back to last directory
JohnDoe@htb[/dev/shm]$ cd -
JohnDoe@htb[~]$

# Go up one level
JohnDoe@htb[/dev/shm]$ cd ..
JohnDoe@htb[/dev]$

# TAB auto-complete
JohnDoe@htb[~]$ cd /dev/s [TAB TAB]
shm/ snd/
💡
Dot notation. = current dir  |  .. = parent dir  |  ~ = home dir

File Management — touch, mkdir, mv, cp
09 / 10

Linux gives you full control over files and directories from the command line — creating, moving, renaming, and copying are all fast one-liners.

Creating Files & Directories
TERMINAL
# Create an empty file
JohnDoe@htb[~]$ touch info.txt

# Create a directory
JohnDoe@htb[~]$ mkdir Storage

# Create nested directories in one shot (-p = parents)
JohnDoe@htb[~]$ mkdir -p Storage/local/user/documents

# Create a file inside a specific path
JohnDoe@htb[~]$ touch ./Storage/local/user/userinfo.txt

# Visualize the directory tree
JohnDoe@htb[~]$ tree .
.
├── info.txt
└── Storage/local/user/
├── documents
└── userinfo.txt
mv — Move & Rename · cp — Copy
TERMINAL — mv syntax: mv <src> <dest>
# Rename a file
JohnDoe@htb[~]$ mv info.txt information.txt

# Move multiple files into a directory
JohnDoe@htb[~]$ touch readme.txt
JohnDoe@htb[~]$ mv information.txt readme.txt Storage/

# Copy a file (original stays)
JohnDoe@htb[~]$ cp Storage/readme.txt Storage/local/
⚠️
mv vs cpmv moves (source is deleted). cp copies (source remains). Always double-check before moving important files.

Terminal Shortcuts & Study Tips
10 / 10

Mastering shortcuts makes you significantly faster — especially during CTF challenges with time pressure.

TERMINAL
# Navigate and clear in one command
JohnDoe@htb[~]$ cd /dev/shm && clear
Essential Shortcuts
Ctrl+LClear terminal
Ctrl+RSearch command history
↑/↓Scroll history
TABAuto-complete
TAB TABShow all completions
Ctrl+CKill process
Ctrl+ZSuspend process
📖
Use man pagesType man <command> for the full manual of any command. You'll often find powerful flags you didn't know existed.
🧪
Experiment on your VMTake a snapshot first, then break things intentionally. Fixing what you broke is the fastest way to learn.
🔍
Think like an attackerFor every command you learn, ask: how would this be used during a pentest? Commands like uname -r, id, ip a, and ls -la are among the first run after gaining access.
