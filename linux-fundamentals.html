Connecting via SSH
01 / 09

SSH (Secure Shell) lets you remotely access machines securely over a network — the standard way to connect to any remote Linux system.

TERMINAL
$ ssh user@<TARGET_IP>
user@<TARGET_IP>'s password: ••••••••
Welcome to Ubuntu 20.04 LTS...
JohnDoe@target:~$
💡
TipReplace <TARGET_IP> with the actual IP address of the machine you are connecting to.

hostname — Identify the Machine
02 / 09

The hostname command prints the name of the computer you are currently logged into. Useful for confirming which machine you have accessed.

TERMINAL
JohnDoe@target[~]$ hostname
linux-target-01

whoami — Current User
03 / 09

whoami works on both Windows and Linux. After gaining access to a system this is one of the first commands to run — knowing which user you are running as tells you what you can and cannot do on that system.

TERMINAL
JohnDoe@target[~]$ whoami
JohnDoe
⚠️
Why it mattersYour current user determines privileges and access on the system — a critical first step before attempting any further enumeration or privilege escalation.
🪪
id — User Identity & Group Membership
04 / 09

id expands on whoami by showing your effective group memberships and IDs — invaluable for understanding what permissions and access you actually have on a system.

TERMINAL
JohnDoe@target[~]$ id
uid=1000(JohnDoe) gid=1000(JohnDoe) groups=1000(JohnDoe),
4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),
116(lpadmin),126(sambashare)
GroupWhat it means
adm (4)Can read logs in /var/log — potential access to sensitive info
sudo (27)🚨 Can run commands as root — key privilege escalation path
any custom groupAlways investigate non-standard groups — they may grant access to files or services

ip — Network Interfaces & Routing
05 / 09

The ip command shows and manipulates routing, network devices, interfaces, and tunnels. It is the modern replacement for ifconfig and is standard on all current Linux distributions.

Each network interface has an MTU (Maximum Transmission Unit) — the largest packet size it can transmit. Standard Ethernet interfaces use an MTU of 1500 bytes. The loopback interface (lo) uses 65536. Knowing the MTU of each interface is useful during network enumeration and troubleshooting.

TERMINAL — ip a (show all interfaces)
JohnDoe@target[~]$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN
link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
inet 127.0.0.1/8 scope host lo
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast
link/ether 00:0c:29:xx:xx:xx brd ff:ff:ff:ff:ff:ff
inet 10.10.x.x/24 scope global eth0
TERMINAL — filter to MTU lines only
JohnDoe@target[~]$ ip a | grep mtu
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 ...
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ...

# Link-layer view only (no IP addresses shown):
JohnDoe@target[~]$ ip link
CommandWhat it shows
ip aAll interfaces with IP addresses, MTU, and state
ip addrSame as ip a (full form)
ip linkLink-layer info (MAC, MTU, flags) — no IP addresses
ip a | grep mtuShow only MTU lines for all interfaces
ip routeRouting table — useful for identifying gateways and network segments

uname — Kernel & System Info
06 / 09

uname prints information about the operating system and kernel. The most useful flags are -a (all info at once) and -r (kernel release version alone).

TERMINAL
JohnDoe@target[~]$ uname -a
Linux target 4.15.0-99-generic #100-Ubuntu SMP Wed Apr 22 20:32:56 UTC 2020 x86_64 GNU/Linux

JohnDoe@target[~]$ uname -r
4.15.0-99-generic
FlagOutput
uname -aAll info — kernel name, hostname, release, version, arch, OS
uname -rKernel release version — useful for researching known vulnerabilities
uname -sKernel name (e.g. Linux)
uname -nNetwork node hostname
uname -mMachine hardware architecture (e.g. x86_64)
uname -oOperating system (e.g. GNU/Linux)
💡
Kernel version researchTake the output of uname -r and search it alongside terms like "exploit" or "privilege escalation" to quickly identify publicly known kernel vulnerabilities for that exact version.

Navigation — pwd, ls, cd
07 / 09

Navigation in Linux is done entirely through the terminal. Confirm where you are with pwd, list contents with ls, and move with cd.

pwd — Print Working Directory
TERMINAL
JohnDoe@target[~]$ pwd
/home/JohnDoe
ls — List Directory Contents
TERMINAL — ls variants
# Basic list
JohnDoe@target[~]$ ls
Desktop Documents Downloads Music Pictures Public Templates Videos

# Long format — permissions, owner, size, date
JohnDoe@target[~]$ ls -l
drwxr-xr-x 2 JohnDoe users 4096 Nov 13 17:37 Desktop
drwxr-xr-x 2 JohnDoe users 4096 Nov 13 17:34 Documents ...

# Long format + hidden files (filenames starting with .)
JohnDoe@target[~]$ ls -la
.bash_history .bashrc Desktop Documents ...

# Include inode numbers (-i) — unique filesystem ID per file
JohnDoe@target[~]$ ls -i
262401 Desktop 131074 Documents 655362 Downloads

# Inode + full detail + hidden files — maximum info
JohnDoe@target[~]$ ls -ila
262401 drwxr-xr-x 2 JohnDoe users 4096 Nov 13 17:37 Desktop
131074 drwxr-xr-x 2 JohnDoe users 4096 Nov 13 17:34 Documents

# List a specific path without navigating there first
JohnDoe@target[~]$ ls -l /var/
ls -l Column Breakdown
ColumnMeaning
drwxr-xr-xFile type (d=dir, -=file) + permissions for owner / group / others
2Number of hard links
JohnDoeOwner
usersGroup owner
4096Size in bytes
Nov 13 17:37Last modified timestamp
💡
Inodes explainedEvery file and directory on a Linux filesystem has an inode — a unique index number the OS uses to track it regardless of filename. The -i flag on ls reveals it. Use stat <file> for even more inode detail including block count and all timestamps.
cd — Change Directory
TERMINAL — cd variants
# Navigate to an absolute path
JohnDoe@target[~]$ cd /dev/shm
JohnDoe@target[/dev/shm]$

# Jump back to the previous directory
JohnDoe@target[/dev/shm]$ cd -
JohnDoe@target[~]$

# Go up one level to the parent directory
JohnDoe@target[/dev/shm]$ cd ..
JohnDoe@target[/dev]$

# TAB auto-complete — press twice to see all options
JohnDoe@target[~]$ cd /dev/s [TAB TAB]
shm/ snd/
💡
Dot notation. = current directory  |  .. = parent directory  |  ~ = home directory

File Management — touch, mkdir, mv, cp
08 / 09

Linux gives you full control over files and directories from the command line — creating, moving, renaming, and copying are all fast one-liners.

Creating Files & Directories
TERMINAL
# Create an empty file
JohnDoe@target[~]$ touch info.txt

# Create a directory
JohnDoe@target[~]$ mkdir Storage

# Create nested directories in one shot (-p creates parents)
JohnDoe@target[~]$ mkdir -p Storage/local/user/documents

# Create a file at a specific path
JohnDoe@target[~]$ touch ./Storage/local/user/userinfo.txt

# Visualize the full directory tree
JohnDoe@target[~]$ tree .
.
├── info.txt
└── Storage/local/user/
├── documents
└── userinfo.txt
mv — Move & Rename · cp — Copy
TERMINAL — mv syntax: mv <src> <dest>
# Rename a file
JohnDoe@target[~]$ mv info.txt information.txt

# Move multiple files into a directory at once
JohnDoe@target[~]$ touch readme.txt
JohnDoe@target[~]$ mv information.txt readme.txt Storage/

# Copy a file — original stays in place
JohnDoe@target[~]$ cp Storage/readme.txt Storage/local/
⚠️
mv vs cpmv moves the file (source is deleted). cp copies it (source remains). Always double-check your destination before running mv.

Terminal Shortcuts & Study Tips
09 / 09

Mastering these shortcuts will make you significantly faster in the terminal — every second saved adds up during time-pressured work.

TERMINAL
# Chain commands — navigate and clear in one line
JohnDoe@target[~]$ cd /dev/shm && clear
Essential Shortcuts
Ctrl+LClear terminal
Ctrl+RSearch command history
↑/↓Scroll history
TABAuto-complete
TAB TABShow all completions
Ctrl+CKill current process
Ctrl+ZSuspend current process
📖
Use man pagesType man <command> for the full manual of any command. You will often find powerful flags you had no idea existed.
🧪
Experiment on a VMSnapshot your VM first, then break things intentionally. Fixing what you broke is the fastest way to truly understand how something works.
🔍
Think like an attackerFor every command you learn, ask: how would this be used during an engagement? Commands like uname -r, id, ip a, and ls -la are among the first things run after gaining access to a new system.
