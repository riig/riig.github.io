ip — Network Interfaces & Routing
06 / 10

The ip command shows and manipulates routing, network devices, interfaces, and tunnels. It is the modern replacement for ifconfig and is standard on all current Linux distributions.

Each network interface has an MTU (Maximum Transmission Unit) — the largest packet size it can transmit. Standard Ethernet interfaces use an MTU of 1500 bytes. The loopback interface (lo) uses 65536.

TERMINAL
JohnDoe@target[~]$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN
inet 127.0.0.1/8 scope host lo
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast
inet 10.10.x.x/24 scope global eth0

JohnDoe@target[~]$ ip a | grep mtu
1: lo: ... mtu 65536 ... 2: eth0: ... mtu 1500 ...
CommandWhat it shows
ip aAll interfaces with IP addresses, MTU, and state
ip linkLink-layer info (MAC, MTU, flags) — no IP addresses
ip a | grep mtuShow only MTU lines for quick comparison
ip routeRouting table — find gateways and network segments

10 / 10

Once on a system you need to know how to find things — tools, configuration files, recently modified files. Three commands handle most of these cases.

which — Locate a Binary
TERMINAL

find — Deep Filesystem Search
TERMINAL — combining multiple filters
# Files owned by root, larger than 20KB, modified after 2020-03-03
JohnDoe@target[~]$ find / -type f -name *.conf -user root -size +20k -newermt 2020-03-03 -exec ls -al {} \; 2>/dev/null
-rw-r--r-- 1 root root 136392 Apr 25 /usr/src/linux-headers-5.5.0/include/config/auto.conf
-rw-r--r-- 1 root root 95813 May 7 /usr/share/metasploit-framework/data/jtr/repeats32.conf
...
OptionMeaning
-type fFiles only (-type d for directories)
-name *.confMatch filename pattern (asterisk = wildcard)
-user rootOwner must be root
-size +20kLarger than 20KB (-size -25k = smaller than 25KB)
-newermt DATEModified after specified date
-exec ls -al {} \;Run ls -al on every result
2>/dev/nullSuppress permission-denied errors
locate — Fast Index Search
TERMINAL
# Update the database first, then search
JohnDoe@target[~]$ sudo updatedb
JohnDoe@target[~]$ locate *.conf
/etc/GeoIP.conf
/etc/NetworkManager/NetworkManager.conf
/etc/adduser.conf ...
💡
find vs locatelocate is much faster but searches a pre-built database that may be stale. find searches the live filesystem and supports complex filters. Use locate for speed, find for precision.

File Descriptors & Redirections
11 / 20

A file descriptor (FD) is a reference maintained by the kernel to manage Input/Output operations. Think of it like a ticket number that connects a process to an open file, socket, or I/O resource. By default Linux provides three standard file descriptors:

STDIN  —  FD 0 STDOUT  —  FD 1 STDERR  —  FD 2
TERMINAL — redirect STDERR to suppress errors
# Without suppression — errors show alongside results
JohnDoe@target[~]$ find /etc/ -name shadow
/etc/shadow
find: '/etc/ssl/private': Permission denied

# Redirect STDERR (FD 2) to null — errors discarded
JohnDoe@target[~]$ find /etc/ -name shadow 2>/dev/null
/etc/shadow

# Redirect STDOUT to file, STDERR to separate file
JohnDoe@target[~]$ find /etc/ -name shadow 2> stderr.txt 1> stdout.txt
TERMINAL — output redirection operators
# Overwrite file with output
JohnDoe@target[~]$ find /etc/ -name shadow 2>/dev/null > results.txt

# APPEND to existing file (>> instead of >)
JohnDoe@target[~]$ find /etc/ -name passwd >> results.txt 2>/dev/null

# Use file as STDIN with <
JohnDoe@target[~]$ cat < results.txt
/etc/shadow
/etc/passwd

# Heredoc — feed multiline text into command
JohnDoe@target[~]$ cat << EOF > stream.txt
> Line one
> Line two
> EOF
OperatorWhat it does
>Redirect STDOUT to file — overwrites
>>Redirect STDOUT to file — appends
2>/dev/nullDiscard STDERR (hide errors)
2> fileRedirect STDERR to a file
<Use file content as STDIN
<< EOFFeed multiline heredoc as STDIN
|Pipe STDOUT of one command to STDIN of next
Pipes
TERMINAL — chaining commands with pipes
# Find .conf files in /etc, filter for systemd, count results
JohnDoe@target[~]$ find /etc/ -name *.conf 2>/dev/null | grep systemd | wc -l
6
💡
Pipes are limitlessYou can chain as many pipe stages as you need. Each | passes the STDOUT of the left command into the STDIN of the right command. This is the foundation of Linux command-line power.

# 1. cut — extract field 1 (username) using : as delimiter

root sync postgres mrb3n cr4y0n JohnDoe

# 2. tr — replace : with space for readable output
JohnDoe@target[~]$ cat /etc/passwd | grep -v "false\|nologin" | tr ":" " "

# 3. column -t — align into tidy table
JohnDoe@target[~]$ ... | tr ":" " " | column -t
root x 0 0 root /root /bin/bash
JohnDoe x 1002 1002 JohnDoe /home/JohnDoe /bin/bash

# 4. awk — print only first and last fields
JohnDoe@target[~]$ ... | awk '{print $1, $NF}'
root /bin/bash JohnDoe /bin/bash

# 5. sed — substitute text (s/find/replace/g)
JohnDoe@target[~]$ ... | sed 's/bin/BOX/g'

# 6. wc -l — count total matches
JohnDoe@target[~]$ ... | wc -l
6
🔗
The full pipelinecat /etc/passwd | grep -v "false\|nologin" | tr ":" " " | awk '{print $1, $NF}' | wc -l — this single line filters, reformats, extracts, and counts in one shot. This is how experienced Linux users work.

Regular Expressions (RegEx)
13 / 20

Regular expressions (RegEx) are patterns used to search, match, and manipulate text. They are available in grep, sed, awk, and most programming languages — making them one of the most universally useful skills in cybersecurity.

At the core of RegEx are metacharacters — symbols that define the structure of a search rather than matching literal characters.

Grouping Operators
OperatorDescription
(a)Round brackets — group parts of a pattern together
[a-z]Square brackets — character class (any char in range)
{1,10}Curly brackets — quantifier (how many times pattern repeats)
|OR operator — match either left or right expression
.*AND-like — both expressions must appear in order
OR and AND Operators with grep -E
TERMINAL
# OR — lines containing "my" or "false"
JohnDoe@target[~]$ grep -E "(my|false)" /etc/passwd
lxd:x:105:65534::/var/lib/lxd/:/bin/false
mysql:x:116:120:MySQL Server,,:/nonexistent:/bin/false

# AND — lines containing both "my" AND "false" in that order
JohnDoe@target[~]$ grep -E "(my.*false)" /etc/passwd
mysql:x:116:120:MySQL Server,,:/nonexistent:/bin/false

# Equivalent using two greps piped together
JohnDoe@target[~]$ grep -E "my" /etc/passwd | grep -E "false"
mysql:x:116:120:MySQL Server,,:/nonexistent:/bin/false
💡
-E for extended regexAlways use grep -E when using operators like |, +, ?, or grouping with (). Without -E these characters are treated as literal text.

Permission Management — chmod, chown, SUID, Sticky Bit
14 / 20

Linux permissions control who can read (r), write (w), and execute (x) files and directories. They are assigned separately for the owner, group, and others.

Reading Permissions
-rwxrw-r-- 1 root root 1641 May 4 23:42 /etc/passwd

│ │││ │││ │││
│ │││ │││ └── Others: r-- (read only)
│ │││ └── Group: rw- (read + write)
│ └── Owner: rwx (read + write + execute)
└── Type: - (file)  d (dir)  l (link)
chmod — Change Permissions
TERMINAL
# Add read permission for all users (symbolic)
JohnDoe@target[~]$ chmod a+r shell && ls -l shell
-rwxr-xr-x 1 JohnDoe JohnDoe 0 May 4 22:12 shell

# Set permissions using octal notation
JohnDoe@target[~]$ chmod 754 shell && ls -l shell
-rwxr-xr-- 1 JohnDoe JohnDoe 0 May 4 22:12 shell
Octal Notation Reference
OctalBinaryPermissionsMeaning
7111rwxRead + Write + Execute
6110rw-Read + Write
5101r-xRead + Execute
4100r--Read only
0000---No permissions
chown — Change Owner
TERMINAL
# Syntax: chown user:group file
JohnDoe@target[~]$ chown root:root shell && ls -l shell
-rwxr-xr-- 1 root root 0 May 4 22:12 shell
SUID & SGID — Elevated Execution Bits

The SUID (Set User ID) and SGID (Set Group ID) bits allow a file to run with the permissions of its owner rather than the user who executes it. They appear as s in place of x in permission strings.

TERMINAL — find SUID binaries on the system
JohnDoe@target[~]$ find / -perm -4000 2>/dev/null
/usr/bin/passwd
/usr/bin/sudo
/usr/bin/pkexec
⚠️
SUID as a privesc vectorIf a program with SUID runs as root and offers shell access (e.g., via ! in a pager), any user can spawn a root shell. GTFObins documents these misconfigurations. Always audit SUID binaries on a system you're assessing.
Sticky Bit — Shared Directory Protection

The sticky bit on a directory ensures that only the file's owner, the directory's owner, or root can delete or rename files inside — even if other users have write access to the directory.

TERMINAL
JohnDoe@target[~]$ ls -l
drw-rw-r-t 3 JohnDoe JohnDoe 4096 Jan 12 12:30 scripts
drw-rw-r-T 3 JohnDoe JohnDoe 4096 Jan 12 12:32 reports
BitMeaning
t (lowercase)Sticky bit set AND execute (x) permission is also set — others can traverse the directory
T (uppercase)Sticky bit set but NO execute (x) — others cannot enter the directory at all
💡
Execute on directoriesExecute permission on a directory means the ability to enter it (cd into it). Without execute, you cannot traverse the directory even if you can see its name — you'll get "Permission denied".
